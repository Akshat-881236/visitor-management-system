Dashboard Access Guide

Learn how secure dashboard access works within the Visitor Management System and understand the responsibilities associated with authenticated access.

Understanding Dashboard Access

The Visitor Management System provides secure dashboard environments for Administrators and Security Personnel. Dashboard access is strictly restricted to authenticated users only. Multiple security mechanisms including credential verification, OTP authentication and session validation are implemented to protect sensitive visitor information and organizational resources.

  • Only authorized Administrators and Security Personnel can access dashboard resources.
  • Users must successfully authenticate using valid credentials before dashboard access is granted.
  • A second verification layer using OTP authentication is mandatory before secure session generation.
  • Successful authentication generates a unique session identifier that is appended to the dashboard URL.
  • Dashboard URLs should never be shared publicly as they contain sensitive authenticated session references.
  • User sessions automatically expire after logout or server restart to prevent unauthorized access.
  • Attempting to access dashboard pages directly without authentication automatically results in access denial.
  • Users must never disclose login credentials, OTP codes or active session identifiers to unauthorized individuals.
  • Dashboards provide access to sensitive visitor records, approvals, check-ins and organizational audit information.
  • Always logout immediately after completing work, especially when using shared or public systems.
  • Dashboard access is optimized for smartphones, tablets, laptops, desktops and large display systems.
  • All dashboard operations are performed under organizational security policies and monitoring procedures.
  • If dashboard access repeatedly fails despite successful authentication, contact the system administrator.
