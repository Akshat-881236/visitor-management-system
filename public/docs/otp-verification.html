Understanding OTP & Two-Tier Authentication

The Visitor Management System utilizes One-Time Password (OTP) based Two-Tier Authentication to protect administrative and security operations. Sensitive operations such as approving visitors, monitoring check-ins, retrieving visitor records and accessing dashboards require strong identity verification. OTP verification acts as an additional security layer that significantly reduces the risk of unauthorized access and credential misuse.

  • What is OTP? OTP stands for One-Time Password. It is a temporary numeric security code generated uniquely for a single authentication request.
  • What is Two-Tier Verification? Two-Tier Verification requires users to successfully complete two independent security checks before obtaining access.
  • First Verification Layer: The user must provide valid credentials such as Email ID, Employee ID or Password.
  • Second Verification Layer: A secure OTP is sent to the registered email address for identity confirmation.
  • Why is OTP Required? Even if passwords are accidentally exposed, attackers cannot access the system without the registered user's email account.
  • OTP is always delivered only to the officially registered email address associated with the authenticated account.
  • Each OTP remains valid only for a limited period to minimize security risks and prevent unauthorized reuse.
  • Expired OTPs cannot be reused. Users must initiate a fresh login request to generate a new verification code.
  • Verification Workflow: Login Request → Credential Validation → OTP Generation → Email Delivery → OTP Verification → Secure Session Generation → Dashboard Access.
  • Users should never share OTP codes with colleagues, visitors or third parties under any circumstances.
  • Upon successful OTP verification, a secure session identifier is generated to maintain authenticated communication.
  • Administrators and Security Personnel should always logout after completing operational activities, particularly on shared devices.
  • If OTP delivery or verification repeatedly fails, users should consult the OTP Troubleshooting documentation or contact system administrators.
  • Can an OTP be reused? No. Every OTP is single-use and automatically invalidated after successful verification or expiration.
  • Can someone login using only my password? No. The system additionally requires OTP verification through the registered email account.
