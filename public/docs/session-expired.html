Understanding Session Expiration

The Visitor Management System implements secure session management to protect sensitive visitor information and organizational resources. For security purposes, every authenticated session automatically expires after eight hours or when the user logs out manually. Once a session expires, users must authenticate again to continue accessing protected dashboard resources.

  • Every authenticated Administrator and Security session remains active for a maximum duration of 8 Hours.
  • After eight hours, the system automatically invalidates the session to prevent unauthorized long-term access.
  • Logging out manually immediately terminates the active session regardless of remaining session time.
  • Server restarts, maintenance activities or security updates may invalidate active sessions before the eight-hour limit.
  • Previously saved dashboard URLs containing old session identifiers cannot be reused after expiration.
  • Session expiration significantly reduces the risk of unauthorized access from unattended or shared devices.
  • Users working on public or shared systems should always logout immediately after completing their tasks.
  • To regain access after expiration, return to the login portal and complete the authentication process again.
  • A fresh OTP verification may be required during re-authentication depending upon organizational security policies.
  • Never attempt to modify session identifiers manually in dashboard URLs.
  • If session expiration repeatedly occurs unexpectedly, contact your system administrator for assistance.
